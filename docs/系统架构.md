# 系统架构文档

> 开发内部文档 · v0.1

---

## 整体概述

基于 Web 的学习资料管理工具。用户上传课程文件，系统自动分类、解析内容、提取关键信息、生成摘要与课程目标总览板，结果统一展示在前端课程空间中。

---

## 技术栈

| 层级 | 技术 |
|---|---|
| 前端 | Next.js · React · TypeScript |
| 后端 | FastAPI · Python |
| 数据库 | PostgreSQL / Supabase |
| 文件存储 | Supabase Storage / AWS S3 |
| 文件解析 | PyMuPDF（PDF）· python-pptx（PPT）· python-docx（Word） |
| 人工智能 | 大语言模型接口（待定） |

---

## 组件职责

### 前端
- 用户登录与身份管理
- 课程空间创建与管理
- 文件上传界面
- 课程目标总览板展示
- 文档摘要与关键信息展示
- 课程页面整体呈现

### 后端
- 接收文件上传请求
- 管理文档元数据
- 触发文档解析与人工智能处理流程
- 将处理结果写入数据库并返回前端

### 文件存储
- 存储用户上传的原始文件（PDF、PPT、Word）
- 不做内容处理，仅持久化存储

### 数据库
- 存储所有结构化应用数据（见数据模型）

### 人工智能处理层
- 文档内容解析与分块
- 关键信息提取（成绩比例、截止日期、考试信息）
- 摘要生成
- 课程目标总览板内容生成

---

## 数据模型

```
用户（users）
└── 课程（courses）
    └── 文件（documents）
        ├── 提取结果（extracted_info）
        │   ├── 成绩比例
        │   ├── 截止日期
        │   ├── 考试信息
        │   └── 课程政策
        ├── 摘要（summaries）
        ├── 目标总览板（course_overview）
        │   ├── 课程核心目标
        │   ├── 近期截止日期（含倒计时）
        │   └── 成绩构成
        └── 分块记录（chunks）— 后续语义检索用
```

**向量数据（embeddings）：** 后续版本启用，用于语义搜索与文档问答。

---

## 数据流转

```
用户创建课程空间
        ↓
用户上传文件
        ↓
文件存入云存储
        ↓
后端记录文档元数据至数据库
        ↓
触发人工智能处理 Pipeline
        ↓
  ┌──────────────────────────────┐
  │  人工智能处理 Pipeline         │
  │  1. 文本提取                  │
  │  2. 文本分块（Chunking）       │
  │  3. 关键信息提取               │
  │  4. 摘要生成                  │
  │  5. 课程目标总览板生成          │
  └──────────────────────────────┘
        ↓
处理结果写入数据库
        ↓
前端展示课程页面
```
